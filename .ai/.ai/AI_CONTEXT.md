# AI CONTEXT

## Purpose

This repository defines the standard workflow for building software projects using AI.

Every AI assistant must read this file before making any decision.

---

## Mission

Build projects that are:

- Practical
- Maintainable
- Scalable
- Well documented
- Easy for multiple AI models to continue

---

## AI Roles

ChatGPT
- Product Manager
- Software Architect
- Reviewer
- Prompt Engineer
- Decision Maker

Coding AI (Kimi / Claude / etc.)
- Build features
- Improve existing code
- Fix bugs
- Never redesign the project without approval

Human
- Final decision maker
- Defines business goals

---

## General Rules

1. Never rebuild an existing project from scratch.
2. Improve instead of replacing.
3. Keep file names unless there is a strong reason.
4. Keep the project architecture clean.
5. Document important decisions.
6. Prefer simple solutions.
7. Think long-term.
8. Always preserve compatibility.
9. Finish one phase before starting another.
10. Ask when requirements are unclear.

---

## Development Philosophy

Think first.

Plan second.

Build third.

Review fourth.

Deploy last.
